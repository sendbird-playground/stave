What is Stave

An AI workspace, not just an AI chat

Chat, code, task context, and operational visibility stay in one place so you can keep momentum without juggling terminals, browser tabs, or branch state by hand.

Task-oriented workspaces

Each task keeps its own chat history, branch context, and optional git worktree. Import existing worktrees or spin up isolated experiments without context bleed.

Information panel

Pin Jira, Figma, Confluence, and Slack links beside notes, todos, PRs, and structured fields so task context stays attached to the workspace.

Replay + notifications

Inspect recent turns, request snapshots, and approval events in Session Replay, then jump back from durable notifications when something needs attention.

How it works

From context to iteration

  1. 01

    Open a project

    Stave keeps project, workspace, and task state together and can import existing branch-backed worktrees when you start.

  2. 02

    Attach context

    Drop files and images, let repo-map prewarm codebase context, and pin Jira, Figma, Confluence, or Slack URLs in the workspace information panel.

  3. 03

    Run Claude or Codex

    Chat, edit, and run terminal commands in one layout. Use approvals, plans, and follow-up turns without leaving the workspace.

  4. 04

    Replay and continue

    Use Session Replay and notifications to inspect what happened, jump back into the right task, and keep the loop moving.

Core features

Built for real task flow, not just chat

Dual-provider runtime

Run Claude and Codex in the same project context. Switch models per task without losing state.

Task workspaces

Keep task history isolated and switch across workspaces without losing branch or provider context.

Repo-map context

Prewarm repo awareness in the background so the first turn starts with codebase context instead of manual copy-paste.

Information panel

Store Jira, Figma, Confluence, and Slack URLs next to PRs, notes, todos, saved plans, and custom fields.

Monaco + terminal

Edit code with TypeScript IntelliSense, run commands, and inspect source control from the same window.

Skills system

Use $skill-name in the composer to trigger reusable workflows across global, user, and workspace scopes.

Session Replay + notifications

Inspect recent turns, request snapshots, and approval events, then jump back from durable read/unread notifications.

Themes + local-first

Built-in light/dark modes, curated presets, and installable custom themes, with SQLite-backed persistence that stays on your machine.
